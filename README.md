            인스턴스의 힙 메모리를 사용해 JWT 토큰을 저장하는 InMemoryJwtRegistry는 다음과 같은 한계점이 있다.
            먼저 각 인스턴스가 독립적으로 JWT 토큰을 관리하기 때문에, 사용자가 특정 인스턴스에 로그인한 후 
            다른 인스턴스로 요청을 보낼 경우 해당 인스턴스에서는 사용자의 인증 상태를 알 수 없다. 
            이는 사용자가 여러 인스턴스에 걸쳐 서비스를 이용할 때 일관된 인증을 하지 못하게 된다.
            또 이로 인해 트래픽이 증가해 인스턴스가 추가되거나 제거될 때, JWT 토큰이 분산되어 관리되기 때문에
            확장성에 제약이 존재한다. 따라서 Redis와 같은 중앙 집중식 저장소를 활용해 JWT 토큰을 관리함으로써 이러한 문제를 해결할 수 있다.
